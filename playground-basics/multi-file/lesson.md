# Мультифайловый проект

Playground поддерживает несколько файлов — как настоящий Go-проект с пакетами.

## Один файл vs дерево

| Layout | Когда использовать |
|--------|-------------------|
| `single` | Один main.go, быстрый пример |
| `flat` | 2-3 файла, табами переключаемся |
| `tree` | Пакетная структура, папки |

## Пример: пакет + main

> [!sandbox] go

Попробуйте создать файл `utils/math.go` с функцией `Add(a, b int) int` и вызвать её из `main.go`.

## Как playground обрабатывает файлы

1. Все файлы отправляются на сервер как `[]FileEntry`
2. Сервер раскладывает их по путям в рабочей директории `/work/`
3. Если нет `go.mod` — создаётся автоматически (`module playground`)
4. `go build ./...` компилирует всё дерево

## Файлы read-only

Автор может пометить файл как `readonly: true` — студент видит его, но не может редактировать. Полезно для:
- Тестов (в challenge)
- Готовой инфраструктуры (main с сервером)
- Примеров, которые не нужно менять

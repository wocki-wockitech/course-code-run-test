# SQL Challenge

Отдельный тип challenge — студент пишет SQL-запрос, платформа сравнивает результат с эталоном.

## Как работает

1. Автор даёт `setup.sql` — схема + seed data
2. Студент пишет запрос в `solution.sql`
3. Платформа выполняет оба (эталон + студент) на эфемерной БД
4. Сравнивает result sets

## Режимы сравнения

| Mode | Что проверяет |
|------|--------------|
| `exact_rows` | Точное совпадение (порядок важен) |
| `set_match` | Мультимножество (порядок не важен) |
| `column_subset` | Студент вернул минимум нужные колонки |
